windows,linux,wsl1,wsl2下的命令行代理配置

在日常的命令行使用中，我们有时候需要在命令行使用代理，但是有时又不需要，因此在命令行配置两个命令来控制代理就会显得比较方便

  • spp：应用代理环境变量
  • upp：取消代理环境变量

但是在各个环境下的代理配置又不尽相同，每次要用到的时候还需要去查，因此笔者整理了一下常用系统下命令行代理设置方式，方便查阅。

Windows下powershell7命令行代理配置

基本概念

Powershell中不像bash那样所有东西都是字符串，它支持对象，比如环境变量是存放在Env对象中的，添加和移除环境变量都是操作Env这个对象。

操作对象的方式（以环境变量为例）：

  • 添加/修改一个键值对：Set-Item Env:http_proxy = "http://127.0.0.1:1080"
  • 移除一个键值对：Remove-Item Env:http_proxy

通过这个我们就可以来设置代理了

wsl2下的代理配置

WSL2 基于 Hyper-V 运行，导致 Linux 子系统和 Windows 在网络上是两台各自独立的机器，从 Linux 子系统访问 Windows 首先需要找到 Windows 的 IP。

因此我们在wsl2上的代理设置可以采用局域网代理共享的方式。

两个关键步骤：

  1. WSL2 中配置的代理要指向 Windows 的 IP；

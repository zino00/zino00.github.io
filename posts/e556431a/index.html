论文笔记：软件恶意行为激活相关论文调研

恶意行为激活论文调研

1.多路径探索

论文信息

论文名称:Exploring Multiple Execution Paths for Malware Analysis

时间出处:2007 IEEE

论文概述

首先为程序标记了某些感兴趣的输入，并在执行期间动态地跟踪它们的传播（污点分析）。我们的系统监视这些输入值被过程移动和操作的方式。每当我们检测到基于标记值的控制流决策时，就会存储进程地址空间的当前内容。然后，执行正常进行。当进程以后希望终止时，会自动将其重置为以前存储的快照。这是通过将进程地址空间的当前内容替换为以前存储的值来实现的。此外，我们重写了在控制流决策中使用的输入值，这样这个决策的结果就会颠倒过来。然后，流程沿着另一个分支继续执行。当然，一行中可能会遇到多个分支。在这种情况下，执行空间是通过按深度优先顺序选择延续点来探索的。

当重写某个输入值以探索另一个exe执行路径时，通常仅改变控制流决策使用的单个内存位置是不够的。相反，必须一致地更新(或重写)进程地址空间中与输入相关的所有值。原因是原始的输入值可能已经被复制到其他内存位置，甚至被程序用作以前一些计算的一部分。当只有一个输入实例被修改时，原始值的副本可能会保留在程序的数据段中。这可能导致执行无效操作或探索不可能的路径。因此，为了能够一致地更新和输入及其相关的值，有必要跟踪哪些内存位置依赖于某个输入以及它们如何依赖于这个值。

问题

存在执行路径爆炸和回滚开销大的问题

2.特定领域的多路径探索优化

论文信息

论文名称:RAMBO: Run-time packer Analysis with Multiple Branch Observation

时间出处:DIMVA 2016

论文介绍

运行时打包是恶意软件作者为了隐藏(例如，加密)恶意代码并在运行时恢复它而使用的一种技术。特别是，一些运行时包装器只在需要时解密个别区域的代码，并在它们不运行时重新加密它们。这种技术被称为移动解码帧，它可以极大地复杂化恶意软件分析。

分析这些示例时想到的第一个解决方案是应用多路径探索来触发所有代码区域的解包。不幸的是，众所周知，多路径探索有几个局限性，比如它在分析现实世界的二进制文件时的可伸缩性有限。

在本文中，作者提出了一套特定领域(程序解包领域）的优化和启发式方法来指导多路径探索，并提高其效率和可靠性，以解包的二进制保护移动解码帧。

方法概述

优化和启发式方法的引导目的都是将程序执行引导向未解包区域

优化主要是对符号执行的优化

启发式方法通过提高可以到达未解包区域路径的优先级来引导多路径探索

3.并行环境空间下的推测执行

论文信息

论文名称:GOLDENEYE: Efficiently and Effectively Unveiling Malware’s Targeted Environment

时间出处:RAID 2014

论文介绍

其关键思想是，通过在单个恶意软件执行过程中提供几个动态的、并行的虚拟环境空间，GOLDENEYE通过一个特殊设计的投机执行引擎来观察恶意软件在不同环境下的行为，主动确定恶意软件的目标环境。此外，GOLDENEYE动态地、自适应地切换分析环境，让恶意软件自己暴露其依赖于目标环境的行为。虽然GOLDENEYE用空间来换取速度，但有趣的是，我们的实验结果表明，GOLDENEYE实际上可以使用更少的内存空间，同时实现比现有的多路径探测技术更高的速度。

方法概述

关键设计1——动态构建并行空间来揭露恶意行为

为了克服预先静态指定多个分析环境的局限性，我们的设计是根据恶意软件的行为动态构造多个环境，调用环境查询api。特别是，通过预先标记这些api，我们可以理解每个环境查询的所有可能的返回值。对于每个可能的返回值，我们为其构造一个环境。

例如，如果我们发现恶意软件查询系统调用GetKeyboardLayout，我们可以准备多个返回值，例如中文的0x0004，美国的0x0409，并模拟两个具有中英文键盘的并行运行环境来分析恶意软件的行为。如图2所示，并行环境是与恶意软件的执行一起构建的，因此，它可以防止多次运行相同的样本。只要我们的API标注(第4节介绍)能够涵盖环境查询，我们相信GOLDENEYE能够自动检测/暴露样品的所有环境敏感行为。

关键设计2——在并行环境中应用投机执行

我们应用投机执行[31]的概念,是指当计算机系统执行一些不一定需要的任务在我们的上下文中应用投机执行的优点是尽可能地让执行向前进行。因此，我们考虑在线构建多个可能的环境，并在每个环境实例中推测执行恶意软件。通过确定最可能的恶意执行路径，我们还可以确定为了到达某个路径，运行的环境是什么。

首先，由于在每个环境中执行的指令是不同的，实际上以在线动态方式预测执行是不可行的。我们通过在基本块级别上逐步执行投机执行来解决这个挑战。特别是，我们在所有可选环境设置中执行每个基本块。由于在每个基本块中没有分支指令，所有环境中的指令都是相同的。当我们到达一个块末尾的分支指令时，我们应用几个启发式方法来确定哪个是可能的恶意路径。因此，我们通过只保留最可能通向期望路径的设置来减少空间。

其次，投机执行本质上是速度和空间之间的一种权衡方案(即用更多的内存消耗来换取加速)[31]。在我们的设计中，我们还试图通过两种新颖的设计来减少内存消耗:

  • (1) 我们只推测执行在不同环境下产生不同结果的指令。我们选择采用污染分析，以缩小范围，以操作环境相关数据的说明。
  • (2) 我们监控内存的使用，以防止可选环境的爆炸。

主要步骤

我们在代码块的粒度上进行投机执行，以模拟恶意软件在多个并行环境空间中的执行。

我们首先预取一个指令块。接下来，我们对预取的指令应用污染分析，并将API/指令输出(环境元素)的每个字节污染为污染源。使用污染分析的原因是过滤那些与环境无关的指令，这可以减少完全投机执行的开销。我们传播被污染的标签，当我们使用被污染的操作数命中一条指令时，我们相应地更新所有可选环境中的执行上下文。我们继续这样的传播，直到我们到达块的末尾，这是一个分支指令。对于分支指令，我们确定它是否会受到受污染字节的影响。如果它是一个环境敏感的分支，我们继续下一步，即分支选择和更新。如果不是，投机执行将开始一个新的预取操作。

如果我们知道哪个分支更有可能是恶意软件作者想要的分支，我们可以动态调整环境，让恶意软件只执行一些指定的分支。原则上，当我们发现一个分支属于一个可能的恶意路径时，我们将重新检查可选的环境，并且只选择可以用于执行所需分支的环境。

特点

  • 程序不断向前执行，没有回滚开销，不需要为每个分支存储快照

  • 使用污染分析跳过许多对环境不敏感的分支，不需要探索全部路径

  • 监控内存的使用，以防止可选环境的爆炸。

4.定向符号执行

论文信息

论文名称:Towards Detecting Trigger-based Behavior In Binaries: Uncovering the Correct Environment

时间出处:SEFM 2019

论文介绍

如果一个程序包含无文档记录的、通常是恶意的、仅在特定情况下执行的功能，那么它就会显示出基于触发器的行为。作者使用了定向符号执行来触发这种行为所需的输入和环境。

具体来说，我们提出了克服环境问题和路径选择问题的技术。我们实现了我们的技术，并在一个真正的恶意软件样本上评估了它们的性能，该样本在收到特定的远程命令后会发起拒绝服务攻击。得益于我们的技术，我们的实现能够确定在合理时间内触发恶意行为所需的那些特定命令和所有其他需求。

方法概述

侧重于如何计算正确的环境条件，以便被分析的恶意软件可以触发特定的行为。我们假设人类分析人员有一个特定的程序点，并希望发现触发其执行所需的输入。为此，我们采用了两种技术:

1)符号摘要函数捕获被调用的系统调用的行为，以便将环境数据模型引入到分析中;

2)最短距离符号执行，这是一种路径选择策略，用于引导分析走向所选的程序点。

  • 实现了一个启发式算法来丢弃包含在最短路径计算中可能导致不正确路径的边。

5.识别行为调度程序

论文信息

论文名称:Identifying Behavior Dispatchers for Malware Analysis

时间出处:ASIA CSS 2021

论文介绍

介绍了一个BDHunter系统，它可以有效地识别基于触发器的恶意软件中的行为分配器。我们证明了在恶意软件分析系统中引导程序执行到行为调度程序可以帮助揭示基于触发器的恶意行为。

我们将行为分配器概括为一种常见的控制流模式，可以通过控制流图(CFG)上的模式匹配来识别:

  1. 它包含一组由分支条件定义的条件分支操作;
  2. 一旦条件满足，它就会引导程序执行恶意操作;
  3. 如果匹配失败，它会继续进行下一次检查

方法概述

识别行为分配器的第一步是从恶意软件样本中收集初步信息，包括从动态执行中调用的API调用和从静态分析中获取的控制流图。

然后BDHunter使用两种算法来检测行为分配器:

  • 一种基于控制流模式匹配（静态分析）

    基于模式的方法在恶意软件二进制文件中进行静态搜索，以识别具有调度程序模式的基本块:一个行为调度程序包含一组CondBrOps(条件分支操作），每个CondBrOps将CommVar（公共常量）与一个触发值进行比较。

  • 另一种基于加权API调用（动态分析）

    在基于权重的方法中，BDHunter根据每个API调用构造恶意操作的重要性为其分配一个初始权重，并将这些权重聚合到这些API调用的调用方。如果在动态执行恶意软件样本时可以收集可疑的API调用，BDHunter将利用这两种方法生成一组候选行为调度程序

    BDHunter在动态执行期间收集被调用的API调用，用于其基于权重的标识符。它使用这些调用来帮助识别行为分配器，从而在基于模式的方法之上提高效率和准确性。

    基于权重的方法主要用于增强基于模式的方法。否则，BDHunter使用基于模式的方法生成一组候选行为调度程序。
